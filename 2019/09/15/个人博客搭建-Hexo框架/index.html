    个人博客搭建-Hexo框架

    前言介绍

    博客的文章主流用MarkDown格式文档（自己的笔记也喜欢用这个格式，有道云中）

    个人博客的分来有静态和动态两种：

    1. 静态博客：方便快捷
    2. 动态博客：有前端后端，动态管理，用户管理，功能更强大

    因为个人现在的知识量，所以选择静态博客，考虑到后续学习后是否会改用动态博客，自我感觉应该不会改变，因为动态够用，而且截止到现在了解，好像动态博客成本更高，时间金钱等方面，我只是拿他做一个工具。

    搭建初始化博客

    下载安装git

    1. 打开git的官方网站–https://git-scm.com/
    2. 下载并安装git
    3. 设置git身份

      下载安装Node.js

    4. 打开Nodejs的官方网站–https://nodejs.org/
    5. 下载并安装LTS长期支持版本
    6. 打开命令窗口（Win+R）并输入代码查看是否安装成功node -v和npm -v

    安装Hexo博客框架

    1. 因为国内镜像源延迟问题，安装中国的CNMP（-g为全局安装）
      npm install -g cnpm --registry=https://registry.npm.taobao.org
    2. 检查是否安装成功cnpm -v
    3. 接下来安装hexo博客框架cnpm install -g hexo-cli
    4. 检查是否安装成功hexo -v

    初始化博客

    1. 建立自己的博客文件夹（如果后期想删除直接删除该文件夹就行）
    2. CMD指向该目录，并执行语句hexo init初始化博客

      如果出现下面代码说明没有安装git

    3. 启动博客hexo server就会出现实例地址http://localhost:4000/

    网络部署git插件

    1. 在命令行窗口选择到博客根目录
    2. 安装git部署插件：执行语句cnpm install --save hexo-deployer-git
    3. 设置目录下的_config.yml文件（站点配置文件）冒号后一定空格
    4. 开始上传代码hexo deploy

    装饰博客

    修改主题

      打开主题介绍，根据介绍安装必须插件，执行安装操作（一般是克隆git，修改配置文件）

    操作使用Hexo

    1. 启动博客：hexo server 可以简写成 hexo s
    2. hexo clean #用来清理缓存文件
    3. hexo generate #生成文件
    4. hexo server #运行本地服务器
    5. hexo deploy #部署到服务器
